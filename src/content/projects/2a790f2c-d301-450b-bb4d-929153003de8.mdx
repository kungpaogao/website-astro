# Here: this website


This website is an experiment using [Notion](https://www.notion.so/) as a [CMS](https://en.wikipedia.org/wiki/Content_management_system). This allows me to have a much better editing experience, which should encourage me to write more content and keep the content more up to date.

The process for updating content is easy as editing in Notion and triggering a rebuild/redeploy (at least in theory).

The site is built using [Astro](https://astro.build/), which statically generates the site and helps me minimize the amount of JavaScript that’s sent to the client. Fun fact: everything on this website <i>should</i> work without JavaScript.

You can find the [source code to this site on GitHub](https://github.com/kungpaogao/website-astro). Feel free to file issues if you find a bug.

<br />


Below (the divider) is what I used to test the Notion parser that I “wrote” (copied a lot of code from existing solutions) to convert Notion blocks to Markdown, which is then rendered using Astro’s built-in support for Markdown (underneath the hood, it uses <code>remark-rehype</code>).

## Milestones


Some tasks that I’m proud of completing while building this website:
- Wrote a collection of scripts to parse Notion blocks to Markdown + made sure everything was accessible!
- Attempted to make the website SEO friendly (it still ranks really low in searches rip)
- Iterated a ton on the navigation bar to make it work without JavaScript (and spent a lot of time reverse engineering [apple.com](http://apple.com/)’s nav bar)
- Migrated content architecture to local MDX files
